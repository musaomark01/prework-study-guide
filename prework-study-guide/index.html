HTML

  • element types are commonly used to add content to pages.
  • Text elements
  • Image elements
  • Header elements
  • List elements

  • Common element types that are used to divide and describe a page.
  • Meta-related elements
  • Content division elements
  • Semantic elements

  • The 'head' element contains information, or metadata, that helps the browser to render the page correctly.
  • Elements included within the 'head' are not visible when a user looks at a webpage.
  • The "section" allow us to organize the content on our page.
  • The 'main' tag specifies the main content of a document.
  • The content inside the 'main' element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
  • The only visible element in the 'head' element is the 'title' element.
  • The 'meta' element defines metadata such as the character set, description, keywords, author, and viewport.
  • The 'charset' element sets our character encoding.
  • The 'http-equiv' element sets an HTTP header for content.
  • The 'viewport' element sets the default view for our webpage.
  • The 'body' element contains the content that is shown to the user.
  • The 'ul', 'ol', and 'li' elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The 'img' element contains information about images that are displayed on the webpage.
  • The 'src' attribute is the most important because it defines the location of the image.
  • The 'alt' attribute specifies an alternate text for an image.
  • The 'br' element creates a line of empty space, or a line break, between two blocks of content.
  • the a' element (which stands for anchor) creates links to the same webpage or other webpages.

  • Elements that are nested inside other elements are called child elements.
  • Element refers to the entire element from opening tag to closing tag.
  • Tag refers only to what appears inside the angle brackets.

CSS

  • We use CSS to define style rules for specific elements of an HTML page and format them using a wide range of style options.
  • There are three ways to style a webpage using CSS: inline CSS, internal CSS style sheet, and external CSS style sheet.
  • Inline styles apply to a specific HTML tag, using a style attribute with a CSS rule to style a particular page element only.
  • Internal style sheet contains CSS rules for the webpage in the head section of the HTML file. The rules only apply to that page.
  • external CCS style sheet allows developers to keep all their CSS rules in a separate file, which makes design changes easier.
  • Declarations contain two important components: the CSS property we want to apply and the value of the property.
  • A 'margin' property indicates how much space we want around the outside of an element.
  • 'Padding' property adds space around the content inside an element.
    • The 'rel' attribute specifies the relationship between the current document and the linked document resource. The 'href' attribute specifies the location (URL) of the external resource.

Git

  • The git command must precede every Git statement.
  • git status: show modified files in working directory.
  • The checkout command is to move the working branch to a new branch.
  • The -b flag creates a new branch.
  • The add command adds modifications in the current working branch to the staging area.
  • The -A flag indicates that we want to add all changes.
  • The git commit command with a commit message that describes what the commit contains.
  • The pull command to receive a branch's modifications into the local environment.
  • The origin keyword indicates the source of the pull will be in the GitHub repo.
  • The push command you can create a new remote branch or add new changes to an existing branch.
