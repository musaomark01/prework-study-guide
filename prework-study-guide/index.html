HTML

  • element types are commonly used to add content to pages.
  • Text elements
  • Image elements
  • Header elements
  • List elements

  • Common element types that are used to divide and describe a page.
  • Meta-related elements
  • Content division elements
  • Semantic elements

  • The 'head' element contains information, or metadata, that helps the browser to render the page correctly.
  • Elements included within the 'head' are not visible when a user looks at a webpage.
  • The "section" allow us to organize the content on our page.
  • The 'main' tag specifies the main content of a document.
  • The content inside the
    element should be unique to the document. It should not contain any content that is repeated across documents such as sidebars, navigation links, copyright information, site logos, and search forms.
  • The only visible element in the 'head' element is the 'title' element.
  • The 'meta' element defines metadata such as the character set, description, keywords, author, and viewport.
  • The 'charset' element sets our character encoding.
  • The 'http-equiv' element sets an HTTP header for content.
  • The 'viewport' element sets the default view for our webpage.
  • The 'body' element contains the content that is shown to the user.
  • The 'ul', 'ol', and 'li' elements represent unordered lists, ordered lists, and list items. In essence, these HTML elements represent bulleted or numbered lists.
  • The 'img' element contains information about images that are displayed on the webpage.
  • The 'src' attribute is the most important because it defines the location of the image.
  • The 'alt' attribute specifies an alternate text for an image.
  • The 'br' element creates a line of empty space, or a line break, between two blocks of content.
  • the a' element (which stands for anchor) creates links to the same webpage or other webpages.

  • Elements that are nested inside other elements are called child elements.
  • Element refers to the entire element from opening tag to closing tag.
  • Tag refers only to what appears inside the angle brackets.

  • The -A flag indicates that we want to add all changes.
  • The git commit command with a commit message that describes what the commit contains.
  • The pull command to receive a branch's modifications into the local environment.
  • The origin keyword indicates the source of the pull will be in the GitHub repo.
